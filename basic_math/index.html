What happens if you try to parse the Integer out of a String like "Error 404"? How about "404 Error"? Other combinations of numbers and letters? What needs to be true about the incoming String for it to be parsed correctly?

If a String is entered with letter-characters first, it appears as "NaN", which means "not a number". If numbers are entered before any text characters, then it will parse the numbers up to the beginning of the text. If numeric values exist after the text, they will not appear. For a String to be parsed correctly to an Integer, it must begin with the numeric values.

Learn about the compound assignment operators (like += and -=) -- what problem do these solve and how are they used? What are situations where you might use a compound assignment operator instead of simple assignment (with =)? Which is easier to read: x += 2; or x = x + 2;?

Compound operators are used to assign a value to a given variable, in addition to the current value of the variable. It always modifies the value of the variable based on what it's current value currently is, rather than a complete value replacement. This is useful for tracking running amounts of any sort of value, rather than replacing the number entirely each time.
