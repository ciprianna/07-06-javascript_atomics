In these exercises, we use prompt() to collect information, but most websites that ask for information do not use prompt() -- how do they collect information (what HTML elements do they use)? What might be an advantage to doing it that way instead of with prompt()? Are there any disadvantages? Why might you use prompt()?

Most websites use a form element in html to collect information. An advantage to that could be that a user can easily enter and review several fields of information at once before clicking submit. Another advantage is that validation can be used fairly easily with the form (and there's probably a good way in JavaScript too, but I don't know it yet).

A potential disadvantage to the form is that if you only need one field of information, or something quick, a user has to go to a separate page and input it, then submit. Alternatively, the prompt is very quick without needing the user to navigate anywhere. Prompt might be used if it's just for something short and immediate that the website needs from the user.
